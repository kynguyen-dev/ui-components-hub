## Overview

**Astro** is the foundation of UI Components Hub. As a content-first framework, Astro renders static HTML by default and hydrates interactive components only when needed via "islands architecture."

The Astro library in the hub covers structural, non-interactive patterns that benefit from zero-JS delivery.

## Why Astro Components?

- **Zero JS** — Structural components don't need hydration, so they ship as pure HTML+CSS.
- **Fast page loads** — No framework runtime for layout and background elements.
- **SEO-friendly** — Server-rendered HTML is indexed immediately by search engines.
- **Island-compatible** — Astro components can wrap React islands with `client:load` for selective interactivity.

## Usage in MDX

Astro components can be imported directly in `.astro` files. For MDX usage, wrap them as React components or use them as layout patterns:

```astro
---
import MainLayout from '@/layouts/MainLayout.astro';
---

<MainLayout title="My Page">
  <section class="py-20 text-center">
    <h1>Welcome</h1>
  </section>
</MainLayout>
```
